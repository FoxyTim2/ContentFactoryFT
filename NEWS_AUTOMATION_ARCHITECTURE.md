# Telegram News Relay Bot (MVP)

Этот репозиторий теперь сфокусирован на **Telegram-only** сценарии:
- читаем посты из выбранных Telegram-каналов,
- переводим/редактируем текст на русском,
- публикуем в целевой Telegram-канал,
- сохраняем архитектуру расширяемой (позже можно добавить VK, RSS, сайты и т.д.).

## Что реализовано в текущем MVP
1. Источники: Telegram-каналы (`TG_SOURCE_CHANNELS`).
2. Сбор сообщений за заданный lookback-период.
3. Дедупликация уже обработанных сообщений (SQLite state store).
4. Обработка текста:
   - с OpenAI (перевод + лёгкая редактура),
   - или fallback без LLM (`NoOpContentProcessor`).
5. Публикация в Telegram через Bot API.
6. Циклический polling с настраиваемым интервалом.

## Расширяемость
Текущая структура кода уже разделена по слоям:
- `telegram_client.py` — адаптер источника,
- `llm.py` — адаптер обработки текста,
- `publisher.py` — адаптер публикации,
- `state.py` — слой состояния/дедупликации.

Чтобы добавить другие каналы (VK/RSS/Web), достаточно добавить новые source/publisher-адаптеры и включить их в pipeline.

## Ограничения MVP
- Пока нет админ-панели.
- Нет автоматического фактчека.
- Нет очередей задач (single-process loop).

## Ближайшие шаги
1. Добавить приоритизацию и фильтрацию по ключевым темам.
2. Добавить confidence-score и ручную модерацию.
3. Вынести polling в очередь задач (Celery/RQ) при росте нагрузки.
